# Ecotrax MVP — Implementation Phases

**Spec:** [docs/superpowers/specs/2026-05-03-ecotrax-mvp-design.md](../specs/2026-05-03-ecotrax-mvp-design.md)

The MVP is implemented across 7 sequential plans. Each plan produces working, testable software at its boundary. Plans are written one at a time so each one is informed by what was actually built in the prior plan.

| # | Plan | File | Status | What it ships |
|---|---|---|---|---|
| 1 | Foundation | [`2026-05-03-ecotrax-01-foundation.md`](2026-05-03-ecotrax-01-foundation.md) | **Shipped** (28 commits on `main`) | Next.js 15 + Supabase scaffold, all schema migrations (Layers 1–3), RLS policies, Supabase Auth + role picker, admin seeding, persona route shells, trust UX component scaffolds, GitHub Actions CI, Vercel-ready config. |
| 2 | Source Cache & ETL | [`2026-05-04-ecotrax-02-etl.md`](2026-05-04-ecotrax-02-etl.md) | Drafted | Layer 1 ingestion: GBIF on-demand fetch, IUCN bulk + delta, WDPA shapefile import, GFW alert sync, LandMark ILC import. `species_id_map` reconciliation. GitHub Actions workflows + Vercel Cron. `materialization_jobs` + `source_refresh_log` tracking. **Acceptance:** Each `pnpm run etl:<source>` (and corresponding GH Action) populates Layer 1 with real data; reconciliation produces a populated `species_id_map`. |
| 3 | Integration Layer (Layer 2) | `…03-integration.md` | Pending Plan 2 | `species_profiles` materialization, `species_location_assessments` (eager precomputation for priority set + lazy on-demand), `cost_priors` derivation, hand-curated `intervention_outcomes` seed (50–100 cases), `data_confidence_scores`, `threat_intervention_evidence`, `stakeholder_capacity`, `regulatory_pathways`. pg_cron jobs. **Acceptance:** A query against `species_location_assessments` for a real species + h3_cell returns integrated threat data with provenance and confidence. |
| 4 | AI Layer | `…04-ai.md` | Pending Plan 3 | RAG retrieval bundle builder, prompt versioning system, intervention plan generation with structured output + Zod validator + soft-warn, server-derived confidence, `ai_outputs` audit trail, Q&A capability, funder-project matching, OpenAI embedding pipeline, Vercel AI SDK streaming. **Acceptance:** `POST /api/ai/plan` with a real species + location returns a validated, citation-anchored intervention plan; reproducible from `ai_outputs` row. |
| 5 | Practitioner Experience | `…05-practitioner.md` | Pending Plan 4 | Project Builder wizard (species → location → AI plan → review → submit), org-profile gate, My Projects, outcome reporting form. **Acceptance:** A signed-up practitioner can create, edit, and submit a project end-to-end; submission queues to admin. |
| 6 | Funder Experience | `…06-funder.md` | Pending Plans 4–5 | Marketplace browse + filter + sort, Project Detail with full plan render, pledge flow with off-platform fiscal-sponsor handoff, Portfolio (active/completed/watching), `funder_intent_profiles`, realtime updates. **Acceptance:** A signed-up funder can browse listed projects, view a full plan, and pledge; the practitioner is notified. |
| 7 | Admin Console & Notifications | `…07-admin-notifications.md` | Pending Plans 5–6 | Admin queues (org / project / funding / outcome / AI flag), `admin_actions` log, Resend transactional email, in-app notifications via Supabase Realtime, weekly funder-match digest. **Acceptance:** Admins can move items through every queue; affected users receive email and in-app notifications. |

After Plan 7 ships, the vertical slice is complete per §10 of the spec.
